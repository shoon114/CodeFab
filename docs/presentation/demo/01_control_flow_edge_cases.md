# 데모 1 — 입력 스타일이 달라져도 깨지지 않는 제어문 파싱

## 4-1. 여러 줄(Allman 스타일) 선언

```
var n = 9;
if (n > 3)
{
  print "x";
}
```

출력: `x`. 조건/파라미터와 `{`가 다른 줄에 있어도(`if`/`for`/`func`/`Class`
전부) 정상 동작한다.

## 4-2. 조건+본문이 한 줄에 합쳐진 3단 else-if/else 체인

```
var a = 1;
var b = 5;
if (a > 3) print "x";
else if (b > 1) print "y";
else print "z";
```

출력: `y`

이 입력 스타일은 한때 **"Unexpected token" 구문 오류로 깨졌던 케이스**다.
`else if`도 `else`로 카운트되기 때문에, "이미 한 번 기다렸는지"를 단순
bool 하나로만 추적하면 3단 체인의 마지막 순수 `else`가 짝을 잃어버렸다.
"대기를 시작한 시점의 else 개수"를 정수로 추적해, 개수가 실제로
늘어났을 때만 다시 기다리도록 고쳐서 지금은 두 스타일(줄바꿈/한 줄 결합)
모두 동일하게 동작한다 — 겉보기엔 사소한 입력 스타일 차이가 완전히 다른
버그로 이어질 수 있다는 걸 보여주는 사례.

## 4-3. for 문의 스코프 누수 방지

```
for (var i = 0; i < 3; i = i + 1) { print i; }
print i;
```

출력: `012` 다음 `Undefined variable 'i'` 런타임 에러. `for`의 init에서
선언한 변수는 반복문이 끝나면 확실히 사라진다.
